Git

Управление файлами и папками

  • git init - создает новый проект в текущей директории
  • git init folder-name - создает новый проект в указанной директории
  • git clone URL - клонирует удаленный репозиторий
  • git add . - добавляет в индекс все новые, изменённые, удалённые файлы из текущей директории и её поддиректорий
  • git add text.txt - добавляет в индекс указанный файл (был изменён, был удалён или это новый файл)
  • git rm text.txt - используется в Git для удаления файлов из индекса и рабочей директории
  • git branch название новой ветки - создает новую ветку в репозитории

Приложения и процессы

  • git log - Она выведет список всех наших коммитов.
  • git pull - подтянет изменения из удаленного репозитория в локальный.
  • git push - внесет изменения из локального на удаленный репозиторий
  • git commit -m "message" - Изменения файла сохранятся в репозитории
  • git checkout название ветки, на которую хотим перейти - переключает между ветками

Системная информация

  • git status - показать состояние репозитория (отслеживаемые, изменённые, новые файлы и пр.)
  • git -v - выведет версию Git
  • git config - позволяет просматривать и настраивать параметры, контролирующие все аспекты работы Git, а также его внешний вид.
  • git config --list - показывает все настройки, которые Git найдёт
